# SSUextract 1.1.0

SSUextract 1.1.0 corrects sequence extraction and replaces the legacy reporting
chain with one explicit metadata flow.

## Correctness

Previous versions translated Infernal's 1-based inclusive coordinates into a
Python slice without subtracting one from the lower bound. Every extracted hit was
therefore one nucleotide short. Version 1.1.0 extracts the complete interval on
both strands and verifies the behavior in strand-specific unit tests.

The release also retains FASTA records with descriptions, preserves a final
sequence-map record without a trailing newline, emits valid no-hit reports, and
supports absolute input, model, database, and output paths.

## Pipeline design

The workflow now has four process types: search, extraction, annotation, and final
reporting. Typed hit records provide the shared representation for sequence and
table outputs. BLAST files are scanned once, and final reports are sorted so that
identical inputs produce byte-stable tables.

Category summaries count each contig at most once per category and sample.
Categories assigned by different covariance models are unioned before counting.

Five redundant processing scripts and the legacy temporary-directory layout have
been removed. Input assemblies are no longer republished as pipeline outputs.

## Compatibility

The correction changes extracted sequences by one nucleotide and can change the
best BLAST annotation near a scoring boundary. Coordinates and biological hit
regions remain those reported by Infernal. Consumers that depended on the
off-by-one sequences should regenerate their results.

SSUextract 1.1.0 requires Nextflow 25.04 or later. The tracked `pixi.lock` records
the tested dependency set.

## Verification

The release test suite covers hit parsing, fragment assembly, both strands,
FASTA identifiers, missing sequences, summaries, annotation ranking, database
profiles, taxonomy calibration, configuration parsing, and version consistency.
Two self-contained pipeline integrations cover absolute paths, valid no-hit
input, marker-specific database routing, and rejection of invalid fallback.

The full bundled example produced the same 20 sample/model/contig/coordinate hit
keys as version 0.9.0. Every new sequence is one nucleotide longer, and all 20
old sequences match the appropriate strand-specific substring of the corrected
sequence. Category counts are unchanged. Two best-hit species labels changed
from *Pseudomonas kribbensis* to *Pseudomonas fluorescens* after correcting the
query sequence; both remain in the Bacteria category.

Detailed summaries, category summaries, merged BLAST output, and all extracted
FASTA files were byte-stable across four 1.1.0 trials.
